## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline gaps between badge and text |
| sm | 8px | Counter left padding (`pl-1` = 4px; counter top margin `mt-1` = 4px) |
| md | 16px | Default element padding, banner horizontal padding (`px-3` = 12px rounds to 16px column) |
| lg | 24px | Modal internal padding (`sm:max-w-md` default padding) |
| xl | 32px | Gap between modal sections |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Counter top margin: `mt-1` (4px) — sits close to the input bar without visual weight
- Banner bottom margin: `mb-2` (8px) — tight coupling to the input bar it precedes
- Touch targets: minimum 44px height for the banner dismiss X button and modal buttons (enforced via `h-7` minimum on small buttons, `w-full` on modal CTA)

## Interaction Contract

### Counter — UsageCounter

| Condition | Display | Color |
|-----------|---------|-------|
| `messagesRemaining === null` | Render nothing (prevents flash before fetch) | — |
| `userTier === 'founder'` | "Unlimited messages" | `text-muted-foreground/60` |
| `userTier === 'starter'` | "Unlimited messages" (treat same as founder until Phase 4) | `text-muted-foreground/60` |
| `messagesRemaining >= 6` | "X of 10 messages remaining this month" | `text-green-600 dark:text-green-400` |
| `messagesRemaining === 5` | "5 of 10 messages remaining this month" | `text-amber-500 dark:text-amber-400` |
| `messagesRemaining <= 3` (and > 0) | "X of 10 messages remaining this month" | `text-red-500 dark:text-red-400` |
| `messagesRemaining === 0` | "0 of 10 messages remaining this month" | `text-red-500 dark:text-red-400` |

Counter is always visible for free-tier users. Never dismissible.

### Warning Banner — WarningBanner

| Property | Value |
|----------|-------|
| Trigger | `messagesRemaining !== null && messagesRemaining <= 3` |
| Dismiss | Local `useState(false)` — dismissed flag lives in component, NOT in Zustand. Resets on page reload (by design). |
| Animation in | `animate-in slide-in-from-bottom-2 duration-200` (via `tw-animate-css`) |
| Animation out | No exit animation — component unmounts when `!shouldShow` |
| Placement | Immediately above `<ChatInput>` in both render sites inside `ChatContainer` |
| Icon | `AlertTriangle` from lucide-react, `h-4 w-4 shrink-0` |
| Upgrade button | `variant="outline" size="sm"` — `h-7 text-xs` — links to `href="/pricing"` |
| Dismiss button | Plain `<button>` with `X` icon — NOT a shadcn Button (no ripple/padding needed) |
| Banner width | `w-full max-w-3xl mx-auto` — matches input bar width |
| Border radius | `rounded-xl` — matches glassmorphism input container |

### Exhaustion Modal — ExhaustionModal

| Property | Value |
|----------|-------|
| Trigger | `showExhaustionModal === true` in Zustand store — set when `/api/chat` returns HTTP 429 |
| Does NOT trigger | On page load when `messagesRemaining === 0` (only fires on send attempt) |
| Close behavior | `onOpenChange` sets `setShowExhaustionModal(false)` — also triggered by Esc key via shadcn Dialog |
| Post-dismiss state | Input area becomes disabled (`disabled={true}` passed to `<ChatInput>`) |
| Re-trigger | If user somehow sends again when disabled, the 429 will re-fire and the modal re-opens |
| Input preservation | User's unsent text stays in `ChatInput` local state — no clearing on 429 |
| Conversation preservation | All existing messages in the thread remain visible |
| Modal width | `sm:max-w-md` (shadcn Dialog default) |
| Upgrade button | `variant="default"` — full width — `bg-[#0058be] hover:bg-[#004a9e] text-white` — `href="/pricing"` |
| Dismiss link | Plain `<button>` styled as `text-sm text-muted-foreground hover:text-foreground` |
| Reset date format | `date.toLocaleDateString('en-US', { month: 'long', day: 'numeric', year: 'numeric' })` — e.g. "May 1, 2026" |
| Placement | Once at `ChatContainer` root, outside both welcome/conversation branches — renders in portal |

### Disabled Input State

| Property | Value |
|----------|-------|
| Condition | `messagesRemaining === 0 && userTier === 'free'` |
| Mechanism | Pass `disabled={true}` to both `<ChatInput>` renders in ChatContainer |
| Visual | Existing `disabled` prop already greys the textarea and disables the send button — no new styles needed |
| Placeholder text | "Upgrade to send more messages" — replaces default placeholder when disabled and exhausted |
| Cursor | `cursor-not-allowed` — already applied by existing disabled logic |

## Data Flow Contract

| Step | Action | Implementation |
|------|--------|----------------|
| 1. Mount | ChatContainer `useEffect` fetches `profiles.messages_used_this_period`, `tier`, `period_start` via Supabase browser client | One-time, no deps array |
| 2. Period check | If `period_start` is a prior calendar month, treat `messages_used_this_period` as 0 (client-side lazy reset) | UTC comparison using `getUTCFullYear()` + `getUTCMonth()` — append `T00:00:00Z` to bare date string to avoid timezone trap |
| 3. Store hydration | `setUsageState(used, tier, periodStart)` → sets `messagesRemaining = FREE_TIER_MONTHLY_LIMIT - effectiveUsed` | Founders: store still sets remaining, but UsageCounter ignores the number |
| 4. Send success | `decrementRemaining()` → optimistic local decrement (guard: skip if `messagesRemaining === null` or `userTier === 'founder'`) | Called after successful LLM stream completes |
| 5. 429 received | Parse JSON body from 429 response — set `remaining = 0`, `resetAt = body.reset_at`, `showExhaustionModal = true` | In `handleSendMessage` 429 branch — early return, do NOT throw |
| 6. Voice path | Same 429 detection in `handleVoiceTranscript` — same store updates | Prevents voice mode users from seeing generic error on exhaustion |

State location: all usage state in `useChatStore` (Zustand) — single source of truth. Banner dismiss state only: local `useState` in `WarningBanner` component.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Counter — screen reader | Plain `<p>` element — announced naturally |
| Banner — screen reader | `role="alert"` or rendered inside a live region so it's announced when it slides in |
| Banner dismiss — keyboard | Focusable `<button>` with `aria-label="Dismiss"` |
| Modal — focus trap | Handled by shadcn Dialog (Radix `DialogContent` traps focus) |
| Modal — keyboard close | Esc key closes via Radix Dialog default behavior |
| Modal — screen reader | DialogTitle is read as heading; DialogDescription as body |
| Disabled input | `aria-disabled="true"` passed through existing `disabled` prop chain |
| Color-only distinction | Counter color changes (green/amber/red) are supplemented by the numeric value itself — color is not the only signal |
| Reduced motion | `animate-in` from `tw-animate-css` respects `prefers-reduced-motion` — already handled in `globals.css` |
| Touch targets | Minimum 44px height: modal buttons are full-width; banner dismiss button wraps 16px icon in a comfortable click zone |

## Anti-Patterns

The following patterns are explicitly prohibited in this phase:

| Prohibited Pattern | Correct Approach |
|--------------------|-----------------|
| Fetching profile in `ChatInput` | Fetch in `ChatContainer` — ChatInput is a controlled input, not a data source |
| `sessionStorage` for banner dismiss | `useState` — simpler, matches spec (reappear on refresh) |
| Polling the server for usage updates | One fetch on mount + optimistic decrement only |
| Generic Error throw on 429 | Return early from `handleSendMessage` with modal state set — do NOT propagate to catch block |
| Reading `response.body` stream after calling `response.json()` | These are mutually exclusive on the same Response object — check status before streaming |
| Showing counter or banner for `starter` tier | Treat `starter` same as `founder` (show "Unlimited") until Phase 4 defines starter limits |
